• git init
    git初始化

  • git add .
    监控工作区的状态树，使用它会把工作时的所有变化提交到暂存区，包括文件内容修改(modified)以及新文件(new)，但不包括被删除的文件

  • git add -u
    仅监控已经被add的文件（即tracked file），他会将被修改的文件提交到暂存区。

  • git add -a
    是上面两个功能的合集

  • git diff
    工作区与暂存区或上一次提交进行比较

  • git diff --cached
    比较暂存区与本地库中最近一次提交的内容

  • git status -s
    精简的方式显示文件状态

  • git commit
    提交代码

  • git commit -m 'comment'
    提交代码且注释为"comment"

  • git commit --amend --date="Thu Aug 9 23:04:57 2018 -0700"
    重写最新提交的时间

  • git branch (branch name)
    创建分支

  • git branch -d
    删除分支

  • git branch -r
    查看remote的所有分支

  • git branch -a
    查看本地和remote的所有分支（包括上游项目）

  • git checkout
    切换分支

  • git checkout -b (branch name)
    等价于branch+checkout

  • git checkout -b (branch name) (remote branch name)
    将远程的分支下载后命名为branch name，然后checkout。注意远端的分支和git branch -r里的一致

  • git checkout -t (remote branch name)
    上面的简化版，branch name使用与远端一致的名称

  • git merge -m '' commit名
    合并分支

  • git merge --no-ff
    强制禁用快进模式。详解网页

  • git archive -o filename.zip 版本号
    打包到指定版本的所有代码

  • git log
    输出日志

  • git log --oneline
    简明输出日志，一次commit只显示一行

  • git log --oneline --graph
    附加，显示各个分支之间的关系

  • git show
    显示每次commit的diff之处

  • git show (commit_id)
    显示单次不同之处

  • git tag v1.0
    给HEAD加注释

  • git tag -a v1.0 85fc7e7 -m '1.0版本'
    给版本加带注释的标签

  • git fetch origin master
    github上获取最新版代码，可使用git log -p master..origin/master查看不同之处

  • git pull
    等价于fetch+merge

  • git push
    推送

  • git push origin :分支名
    删除github该分支（要在Settings中修改Default branch为master）

  • git push -u origin master
    设置远程端为origin，将master推入origin

  • git push origin :[tag]
    在github上删除tag

  • git push --amend
    修改最后一次commit的文字

  • git config --system receive.denyNonFastForwards true
    禁止强制更新

  • git config --system receive.denyDeletes true
    规避上述制约可通过删除分支并push新引用。所以加上这一行禁止删除分支和标签

  • touch
    创建空的文本文件

  • git rm
    删除文件

  • git reset --hard <commit_id>
    删除工作空间的改动代码，撤销commit，撤销git add .

  • git reset --soft HEAD^
    不删除工作空间改动代码，撤销commit，不撤销git add .

  • git reset --soft HEAD~2
    不删除工作空间改动代码，撤销两次commit，不撤销git add .

  • git reset --mixed HEAD^
    不删除工作空间改动代码，撤销commit，并且撤销git add .

  • git remote add origin https://github.com/IamWilliamWang/RegexEngine.git
    设置远程git仓库

  • git remote rm origin
    远程仓库置空

  • git remote -v
    查看remote仓库的所有地址（包含上游项目）

  • git remote add upstream *.git
    添加remote仓库的上游项目，比如该项目是从另一个项目fork而来

  • 强制将指针指向远程仓库的HEAD

git reset --hard origin/master
git pull

  • git pull origin master --allow-unrelated-histories
    忽略历史的不同，覆盖本地

  • git stash
    可用来暂存当前正在进行的工作

  • git stash pop
    从Git栈中读取最近一次保存的内容

  • git stash list
    显示Git栈内的所有备份

  • git stash clear
    清空Git栈

  • git stash apply stash@{1}
    可以将你指定版本号为stash@{1}的工作取出来

  • git filter-branch --index-filter 'git rm --cached --ignore-unmatch bin/*.docx' 8f1bf21..HEAD
    从8f1bf21下一个版本开始到HEAD，删除git历史中所有bin/*.docx文件（慎用，操作不可逆）

  • git for-each-ref --format='delete %(refname)' refs/original | git update-ref --stdin
    移除本地仓库中指向旧提交的剩余refs

  • 合并上游项目的commits

git remote add upstream *.git
git fetch upstream
git merge upstream/master
git push origin master
  • 应用.gitignore
git rm -r --cached .       //清空缓存
git add .                  //重新提交
git commit -m ""           //暂存本地
git push                   //推送远端
